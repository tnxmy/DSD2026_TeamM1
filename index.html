Project Overview

Limb Motion Recognition & Assistant

A distributed rehabilitation system developed between UTAD (Portugal) and Jilin University (China). 28 students across 6 teams are building a system that captures limb motion from wearable sensors, processes data on a central server, and delivers real-time feedback through patient and clinician applications.

28
Students across 6 specialized teams
3
System layers: Sensor · Server · Monitor
The system captures limb motion via IMU wearable sensors (S1, S2), processes and classifies movement data through AI inference on a central server (V1, V2), and presents real-time feedback to patients and clinicians through dedicated mobile and web applications (M1, M2). Team M1 is exclusively responsible for the patient-facing mobile application — the Monitor layer's patient endpoint.

System Architecture

M1 Role & Mission

The Patient's Window Into Recovery

Team M1 is responsible for the patient-facing mobile application — the primary interface through which rehabilitation patients receive real-time feedback on their limb movements. We build the experience that turns raw data into clear, reassuring, and actionable guidance.

"The interface is not a layer on top of the system.
For the patient, it is the system."

Fast & Responsive

Low-latency mobile UI that keeps pace with real-time sensor data.

Clear & Intuitive

Motion visualisations designed for patients, not engineers.

Reassuring & Reliable

Consistent alerts and feedback that patients can trust during recovery.

Responsibilities

What We Build

Real-Time Rehabilitation Feedback

Display live motion data from wearable sensors, giving patients immediate feedback during exercises.

Motion Visualisation

Render limb movements in intuitive visual formats so patients can understand their progress.

Posture Correction & Alerts

Detect incorrect postures and alert patients with clear corrective guidance in real time.

Push Notifications

Notify patients of scheduled exercises, progress milestones, and clinician messages.

IF2 Backend Integration

Consume data from the Server layer via the IF2 interface contract, ensuring reliable data flow to the Monitor.
